# 员工工时管理系统功能测试用例

本文档从业务功能角度整理测试用例，适合用于项目验收、面试讲解和后续回归测试。接口级细节见 `test_doc/api-test-record.md` 和 `test_doc/api-doc.md`。

## 1. 测试目标

验证员工工时管理系统的核心功能是否满足需求：

- 管理员可以维护部门、用户、项目等基础数据。
- 部门经理可以为本部门员工进行项目授权。
- 员工可以填报、修改、删除、提交本人已授权项目的工时。
- 部门经理可以审批本部门员工提交的工时。
- 管理员、部门经理、员工可以在各自权限范围内查看工时统计。
- 系统可以记录工时关键操作日志。
- 系统可以限制越权访问和非法数据。

## 2. 测试角色

| 角色 | 主要测试范围 |
| --- | --- |
| 管理员 | 基础数据管理、公司统计、全部日志、权限边界 |
| 部门经理 | 本部门项目授权、本部门工时审批、本部门统计 |
| 员工 | 本人工时填报、修改、删除、提交、个人统计 |

## 13. 回归测试建议

后续每次修改代码后，建议优先回归以下核心链路：

1. 管理员登录，维护部门、用户、项目。
2. 部门经理登录，给本部门员工进行项目授权。
3. 员工登录，新增工时草稿并提交审批。
4. 部门经理登录，审批通过或驳回工时。
5. 管理员登录，查看操作日志和公司统计。
6. 员工登录，查看个人统计和个人中心。

如果这些主流程都正常，说明系统核心业务链路基本稳定。
